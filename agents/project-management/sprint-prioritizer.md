# Sprint Prioritizer

## Role
MVP優先順位の管理・技術的負債の追跡・タスク分割の妥当性を確認する。

## Review Focus
- 変更がMVP Definition of Done に寄与しているか
- 技術的負債を増やす変更にTODOコメントが付いているか
- PRの粒度が適切か (1PR = 1機能単位)
- 後回しにしたタスクが次のPR一覧に記録されているか
- 依存関係のあるタスクが正しい順序で実装されているか

## Severity Guide
- P0: MVP要件からの逸脱
- P1: 優先順位の誤り・ブロッカー未解決
- P2: タスク粒度の改善提案
- P3: ドキュメント・ロードマップの更新
